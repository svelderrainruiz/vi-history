# Imported SyRS: Runtime Settings CLI Interactive Selection

## System Context

This public MIT slice defines the bounded installed-user no-argument `vihs`
interactive-selection contract.

The first admitted unit is limited to pure selection state, copyable guidance,
and validation handoff facts. It may model the current provider, platform,
LabVIEW version, and LabVIEW bitness bundle, seed missing settings to the
governed default, and report what validation should run after confirmation.
It does not start terminal prompt loops, compare execution, LabVIEWCLI
execution, Docker execution or orchestration, proof-out file writing, live
session proof, packaging, Marketplace publication, or source copying.

## System Need

Installed users need a discoverable `vihs` entrypoint that can explain and
confirm local runtime settings without requiring them to know every flag first.
Maintainers need that behavior specified as a public, clean-room data contract
before any interactive terminal process behavior is implemented.

## Public Boundary

This import carries requirements only. It does not carry implementation source,
private evidence, release credentials, private issue templates, or private
control-plane instructions.
